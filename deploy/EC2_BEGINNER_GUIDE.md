# 초보자를 위한 AWS EC2 인스턴스 만들기 (완전 정복)

AWS가 처음이어도 걱정 마세요! 그림을 보듯 자세하게 설명해 드릴게요.

## 1단계: AWS 콘솔 로그인 및 EC2 접속

1.  [AWS 콘솔](https://aws.amazon.com/console/)에 로그인하세요.
2.  왼쪽 위 검색창에 `EC2`라고 입력하고, 검색 결과에서 **EC2**를 클릭하세요.
3.  화면 오른쪽 위에서 **지역(Region)**이 `아시아 태평양(서울) (ap-northeast-2)`로 선택되어 있는지 꼭 확인하세요! (서울이 아니면 속도가 느릴 수 있어요)

## 2단계: 인스턴스 시작 (서버 만들기)

1.  EC2 대시보드 화면 중앙에 있는 주황색 버튼 **[인스턴스 시작]**을 클릭하세요.

### 상세 설정 (천천히 따라오세요)

**1. 이름 및 태그**
*   **이름**: 원하는 이름을 적으세요. (예: `StockNavi-Server`)

**2. 애플리케이션 및 OS 이미지 (Amazon Machine Image)**
*   **Amazon Linux**를 선택하세요. (기본값으로 선택되어 있을 거예요)
*   **Amazon Linux 2023 AMI**라고 적힌 Free tier eligible(프리 티어 사용 가능) 항목인지 확인하세요.

**3. 인스턴스 유형**
*   **t2.micro** 또는 **t3.micro**를 선택하세요.
*   `프리 티어 사용 가능` 표시가 있는 것을 골라야 무료로 쓸 수 있습니다.

**4. 키 페어 (로그인) - 중요! ⭐**
*   "키 페어 이름" 아래에 있는 **[새 키 페어 생성]** 파란색 글씨를 클릭하세요.
*   **키 페어 이름**: `stocknavi-key` (원하는 영어 이름)
*   **키 페어 유형**: `RSA` (기본값)
*   **프라이빗 키 파일 형식**:
    *   Mac/Linux 사용자라면 `.pem`
    *   Windows 사용자인데 `PuTTY`를 쓴다면 `.ppk`
    *   **Tip**: 요즘 Windows 10/11은 터미널(PowerShell)에서 바로 접속 가능하니 **`.pem`**을 추천합니다!
*   **[키 페어 생성]** 버튼을 누르면 파일이 다운로드됩니다.
*   **⚠️ 주의**: 이 파일(`stocknavi-key.pem`)은 잃어버리면 절대 다시 다운로드 못 합니다! **바탕화면이나 찾기 쉬운 곳에 잘 보관하세요.**

**5. 네트워크 설정**
*   오른쪽의 **[편집]** 버튼을 누르세요.
*   **보안 그룹 생성**을 선택합니다.
*   다음 체크박스들을 모두 체크하세요:
    *   ✅ **SSH 트래픽 허용** (내 IP) - *해킹 방지를 위해 '위치 무관'보다는 '내 IP'를 추천하지만, 카페 등에서 하신다면 일단 '위치 무관(0.0.0.0/0)'으로 두셔도 됩니다.*
    *   ✅ **인터넷에서 HTTP 트래픽 허용** (나중에 웹사이트 접속용)
    *   ✅ **인터넷에서 HTTPS 트래픽 허용**

**6. 스토리지 구성 (저장공간)**
*   기본값인 8 GiB로 되어있을 텐데, 프리 티어는 최대 30GB까지 무료입니다. 넉넉하게 **20**으로 바꿔주셔도 좋습니다.

## 3단계: 시작하기

1.  오른쪽 메뉴의 **요약**을 확인하고 주황색 **[인스턴스 시작]** 버튼을 클릭하세요.
2.  "성공적으로 시작했습니다" 메시지가 뜨면 성공입니다!
3.  **[모든 인스턴스 보기]** 버튼을 눌러 목록으로 돌아갑니다.
4.  방금 만든 인스턴스의 상태가 `실행 중 (Running)`이 될 때까지 잠시 기다려주세요 (약 1분 소요).
